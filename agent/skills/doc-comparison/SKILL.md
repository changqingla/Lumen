# 文献对比分析工作流

对多篇学术论文或文档进行多维度结构化对比。

## 触发条件

- 用户要求对比、比较多篇文档
- 用户说"对比一下"、"有什么异同"、"比较这些研究"
- 用户要求分析多篇文献的差异

## 工作流程

### Step 1: 逐篇阅读并按统一维度记录笔记

对每篇文档，按照统一的对比维度提取信息：

1. 查看文档大纲：
```
Action: read_document_outline
Action Input: [doc_id]
```

2. 阅读关键章节：
```
Action: read_document
Action Input: {"doc_id": "[doc_id]", "section": "[关键章节]"}
```

3. 按统一维度记录笔记：
```
Action: write_note
Action Input: {"title": "[文档名] 分析", "content": "研究目标: ...\n研究方法: ...\n数据/样本: ...\n核心发现: ...\n结论: ...\n创新点: ...\n局限性: ..."}
```

### Step 2: 汇总所有笔记

```
Action: read_note
Action Input: all
```

### Step 3: 生成对比分析

```
Action: finish
Action Input: [结构化对比分析]
```

## 输出格式要求

### 对比分析模板

1. **对比概述**：简要说明对比的文献和对比目的

2. **对比维度表格**：

| 维度 | 文献 A | 文献 B | 文献 C |
|------|--------|--------|--------|
| 研究目标 | ... | ... | ... |
| 研究方法 | ... | ... | ... |
| 数据/样本 | ... | ... | ... |
| 核心发现 | ... | ... | ... |
| 创新点 | ... | ... | ... |

3. **相同点分析**：多篇文献的共同之处
4. **差异点分析**：关键差异及其原因
5. **各自优劣势**：每篇文献的独特贡献和不足
6. **综合评价**：整体性的对比结论和建议

## 对比维度参考

根据文献类型灵活选择对比维度：

- **实证研究**：研究问题、假设、方法、样本、变量、结果、效度
- **理论研究**：理论框架、核心概念、论证逻辑、适用范围
- **综述类**：覆盖范围、分类方式、研究空白识别、未来方向
- **技术类**：技术路线、性能指标、适用场景、实现复杂度

## 注意事项

- 所有文献必须使用相同的对比维度，确保可比性
- 表格是核心输出，确保信息准确、简洁
- 差异分析要深入，不要只列举表面差异
- 如果某篇文献缺少某个维度的信息，标注"未涉及"而非编造
