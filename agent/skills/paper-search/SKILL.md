# 论文搜索工作流

通过 MCP 工具搜索学术论文（如 arXiv），并格式化展示结果。

## 触发条件

- 用户要求搜索论文、查找文献
- 用户说"搜索关于XX的论文"、"查找最新研究"
- 用户要求了解某个领域的最新进展

## 前置条件

需要 search_papers MCP 工具可用。如果该工具不可用，告知用户当前未配置论文搜索服务，建议使用 web_search 作为替代。

## 工作流程

### Step 1: 构建搜索查询

根据用户需求构建搜索参数。

**时间敏感搜索的重要规则：**
- "最新论文"：date_from 设为当前日期前 3 个月
- "近期论文"：date_from 设为当前日期前 12 个月
- 绝对不要使用硬编码日期，必须基于当前日期计算

```
Action: search_papers
Action Input: {"query": "[搜索关键词]", "max_results": 10, "date_from": "[计算后的日期，格式 YYYY-MM-DD]"}
```

### Step 2: 格式化展示结果

```
Action: finish
Action Input: [格式化的论文列表]
```

## 输出格式要求

对每篇论文，展示以下信息：

### 论文列表模板

**1. [论文标题]**
- 作者：[作者列表]
- 发表日期：[日期]
- arXiv ID：[ID]
- 摘要：[完整摘要]
- PDF 链接：[URL]
- arXiv 页面：[URL]

**2. [论文标题]**
- ...

### 搜索结果总结

在论文列表后，提供简要总结：
- 搜索到的论文数量
- 主要研究方向/主题分布
- 值得重点关注的论文及理由

## 搜索策略

- 使用英文关键词搜索效果通常更好
- 如果用户用中文描述需求，将关键概念翻译为英文进行搜索
- 如果首次搜索结果不理想，尝试调整关键词或扩大时间范围
- 可以进行多次搜索以覆盖不同角度

## 注意事项

- 始终基于当前日期动态计算 date_from，不要硬编码
- 完整展示摘要，不要截断
- 如果 search_papers 工具不可用，使用 web_search 搜索 Google Scholar 或 arXiv 作为替代
