# 单文档总结工作流

对一篇学术论文或文档进行系统性阅读和结构化总结。

## 触发条件

- 用户要求总结、概述、归纳一篇文档
- 用户说"帮我总结"、"概括一下"、"这篇论文讲了什么"
- 用户关联了一篇文档并要求了解其内容

## 工作流程

### Step 1: 了解文档结构

先用 read_document_outline 查看文档的标题结构和各章节长度，判断文档规模。

```
Action: read_document_outline
Action Input: [doc_id]
```

### Step 2: 阅读文档内容

根据文档长度选择策略：

**短文档（大纲显示总计 < 8000 字符）：**
直接读取全文。

```
Action: read_document
Action Input: {"doc_id": "[doc_id]"}
```

**长文档（> 8000 字符）：**
逐章阅读，每读完一章就用 write_note 记录要点。

```
Action: read_document
Action Input: {"doc_id": "[doc_id]", "section": "[章节关键词]"}
```

然后记录笔记：

```
Action: write_note
Action Input: {"title": "[章节名] 要点", "content": "[该章节的核心内容摘要]"}
```

对每个主要章节重复此过程。

### Step 3: 综合生成总结

**短文档：** 直接基于全文内容生成总结。

**长文档：** 先读取所有笔记，再综合生成。

```
Action: read_note
Action Input: all
```

### Step 4: 输出结构化总结

```
Action: finish
Action Input: [结构化总结]
```

## 输出格式要求

总结应包含以下结构（根据文档类型灵活调整）：

### 学术论文总结模板

1. **研究背景与问题**：该研究要解决什么问题？为什么重要？
2. **研究方法**：采用了什么方法/技术路线？
3. **主要发现/结果**：核心实验结果或理论贡献
4. **结论与意义**：研究的主要结论，对领域的贡献
5. **局限性**（如有）：作者提到的局限或未来工作方向

### 一般文档总结模板

1. **核心主题**：文档的中心议题
2. **关键要点**：3-5 个最重要的观点或信息
3. **详细内容**：按逻辑组织的内容概述
4. **总结**：一段话概括全文

## 注意事项

- 总结语言与用户查询语言保持一致
- 保留关键数据、数字和专业术语
- 不要编造文档中没有的信息
- 对于学术论文，注意区分作者的观点和客观事实
