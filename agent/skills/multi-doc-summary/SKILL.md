# 多文档综合总结工作流

对多篇学术论文或文档进行系统性阅读，按主题综合生成总结报告。

## 触发条件

- 用户关联了多篇文档并要求总结
- 用户说"总结这几篇论文"、"概括这些文献"
- 用户要求对多篇文档进行内容归纳

## 工作流程

### Step 1: 逐篇阅读并记录笔记

对每篇文档重复以下流程：

1. 查看文档大纲：
```
Action: read_document_outline
Action Input: [doc_id]
```

2. 阅读关键章节（重点关注：研究问题、方法、核心发现、结论）：
```
Action: read_document
Action Input: {"doc_id": "[doc_id]", "section": "[关键章节]"}
```

3. 记录该文档的要点笔记：
```
Action: write_note
Action Input: {"title": "[文档名] 要点", "content": "研究问题: ...\n方法: ...\n核心发现: ...\n结论: ..."}
```

### Step 2: 汇总所有笔记

```
Action: read_note
Action Input: all
```

### Step 3: 生成综合总结

```
Action: finish
Action Input: [综合总结报告]
```

## 输出格式要求

综合总结应按主题组织，而非按文档逐篇罗列：

1. **总体概述**：这组文献共同关注的领域/问题
2. **主题分类总结**：
   - 主题 A：涉及的文献及其观点
   - 主题 B：涉及的文献及其观点
   - ...
3. **共同发现**：多篇文献一致的结论或趋势
4. **差异与分歧**：文献之间的不同观点或方法
5. **综合结论**：整体性的总结和启示

## 注意事项

- 按主题组织内容，不要简单拼接各篇文档的独立总结
- 明确标注每个观点来自哪篇文献
- 突出文献之间的共性和差异
- 如果文档数量较多（>5篇），优先关注核心章节，避免迭代次数过多
- 对于每篇文档，笔记应聚焦于：研究问题、方法论、关键数据/结果、结论
